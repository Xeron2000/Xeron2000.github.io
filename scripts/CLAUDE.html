AI 分析系统 ​

模块路径： scripts/
上级导航： PureVex → AI 分析系统

模块概述 ​

AI 分析系统是 PureVex 的智能化核心，负责文章内容的自动分类、标签生成和质量分析，系统基于 Google Gemini AI 模型，为博客内容管理提供智能化的技术支持

核心组件 ​

🛠️ 命令行工具 ​

blog-ai.js - AI 命令行工具 ​

路径： scripts/blog-ai.js
功能： 提供完整的 AI 分析命令行接口

主要命令：

  • analyze - 分析文章并生成 AI 标签
  • reset - 重置文章的 AI 编辑状态
  • status - 查看文章的 AI 编辑状态

使用示例：

bash
# 分析单篇文章
bun run ai analyze --file posts/article.md

# 批量分析文章
bun run ai analyze --dir posts

# 重置 AI 状态
bun run ai reset --dir posts

# 查看分析状态
bun run ai status

🤖 分析引擎 ​

ai-analyzer.mjs - AI 分析核心 ​

路径： .vitepress/utils/ai-analyzer.mjs
功能： 核心分析逻辑实现

核心功能：

  • 文章解析： 提取标题、内容、元数据
  • 分类分析： 智能识别文章类别
  • 标签生成： 基于内容生成相关标签
  • 质量评估： 分析文章质量和可读性

主要函数：

  • batchAnalyzePosts() - 批量分析文章
  • analyzePost() - 分析单篇文章
  • updatePostWithAITags() - 更新文章标签
  • validateTags() - 验证标签格式

⚙️ 配置管理 ​

ai-config.yaml - AI 配置文件 ​

路径： .vitepress/ai-config.yaml
功能： AI 分析引擎的配置管理

配置结构：

yaml
ai:
  enabled: true
  provider: "gemini"
  model: "gemini-1.5-flash"
  
prompts:
  category_analysis: "分类分析提示词"
  tag_analysis: "标签生成提示词"
  
analysis:
  max_content_length: 2000
  confidence_threshold: 0.8
  retry_count: 3

🔄 数据处理流程 ​

  1. 文章读取 - 从文件系统读取 Markdown 文章
  2. 内容解析 - 使用 gray-matter 解析 frontmatter
  3. AI 分析 - 发送内容到 Gemini API 进行分析
  4. 结果处理 - 解析 AI 返回的分类和标签
  5. 数据更新 - 更新文章的元数据信息
  6. 状态记录 - 记录分析时间和状态

🏷️ 标签系统 ​

标签类型：

  • 技术栈标签： python, javascript, typescript
  • 功能标签： tutorial, config, debugging
  • 工具标签： nvim, git, docker

标签生成规则：

  • 使用英文小写
  • 连字符连接多词标签
  • 3-8 个标签每篇文章
  • 优先选择预定义标签

📈 质量控制 ​

置信度阈值： 0.8 重试机制： 最多 3 次 内容长度限制： 2000 字符 标签数量限制： 2-10 个标签

使用指南 ​

🚀 快速开始 ​

1. 环境配置 ​

bash
# 设置环境变量
export GEMINI_API_KEY="your_api_key"
export GEMINI_BASE_URL="https://generativelanguage.googleapis.com"

2. 配置文件 ​

yaml
# .vitepress/ai-config.yaml
ai:
  enabled: true
  provider: "gemini"
  model: "gemini-1.5-flash"
  api_key: "${GEMINI_API_KEY}"

3. 运行分析 ​

bash
# 分析单篇文章
bun run ai analyze --file posts/new-article.md

# 批量分析
bun run ai analyze
